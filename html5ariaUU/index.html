Hvorfor universell utforming?

  • Bruker mer tid på å tenke igjennom hva som faktisk er viktig
  • Lar flere delta i samfunnet og arbeidslivet
  • Mer effektive brukere
  • Enklere selvbetjening

WAI-ARIA

Web Accessibility Initiative – Accessible Rich Internet Applications

  • ARIA standard sier hvordan man kan gjøre ting bra
  • Skjermlesere støtter ikke oppdaterende sider
  • ARIA kan fortelle dette, og sammenhenger

Skjermlesere

<div class="main">
  • Ignoreres
<div class="main" role="main">
  • Bedre, men mye å skrive
<main>
  • Roller er lagt inn i HTML5
  • Støttes av alt før IE 8
<main class="main" role="main">
  • Verden er ikke perfekt

Aria roles

checkbox, button, alert, dialog, menubar, progressbar, ...

  • Forklarer hva denne seksjonen egentlig er

My man tar for gitt

<label for="pwd">
Passord: <input type="password" id="pwd" />
</label>
<span>Passordet må ...</span>
  • Passord: Endre passord

Med aria

<label for="pwd">
Passord: <input type="password" id="pwd" aria-describedby="pwdHint" />
</label>
<span id="pwdHint">Passordet må ...</span>
  • Passord: Endre passord. Passordet må ...
  • States - aria-busy, aria-checked, aria-disabled, ..
  • Required - form om den er required
  • Gir struktur og beskriver metadata
